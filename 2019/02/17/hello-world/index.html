通过github.io快速搭建博客平台是一个免费且快速的选择。借助目前大热的hexo框架(基于node.js)，通过下述步骤就可以20分钟内发布你的第一篇博文啦。

安装Nodejs

无论是apt/yum直接安装，还是利用nvm安装都是可以的。

apt安装方式：

sudo apt-get install nodejs

sudo apt-get install npm 如果没有npm（包管理器），则补充安装

nvm安装方式:

先安装nvm

curl https://raw.github.com/creationix/nvm/master/install.sh | sh

master可以更换成对应的tag版本，会更稳定。

如果安装后输入nvm 显示找不到此命令，则检查bashrc/bash_profile等文件，检查是否有nvm配置，如果没有则手工添加（内容请自行搜索），如果已经存在则source使其在当前bash生效。

之后安装node

nvm install <对应的nodejs版本>

注册Github并且建立博客repo

关于注册，嗯，这个还需要说么？

利用自己的Username，建立一个Username.github.io的仓库。(Username自行替换成自己的。)

安装Hexo

使用之前安装的nodejs包管理器安装hexo

npm install hexo-cli -g

利用Hexo，开始你的博客之旅

  1. 初始化一个Hexo工程

    hexo init <任意工程名>

    _config.yml 记录着本工程的全部配置

    themes 为主题文件夹

    source为源码文件夹

  2. 主题选择

    https://hexo.io/themes/ 上选择你喜欢的主题，并且git clone到hexo工程根目录下themes文件夹，并且修改_config.yml中的theme字段使用你选择的主题。

  3. 编写博客

    在source文件夹下建立博客页面，想简单写就用Markdown就好了(比如本篇)

  4. 测试博客

    hexo server

    访问提示的url即可测试页面

  5. 发布博客

    在发布之前进一步对_config.yml进行配置，详情见https://hexo.io/docs/configuration

    主要需要配置title、author、language、url和deploy 等。

    对于我们要发布到github.io上的博客，deploy配置为

    如果使用git的类型发布的话，还需要安装一个工具：hexo-deployer-git

    npm install hexo-deployer-git --save

    然后就可以正式发布了，

    hexo clean

    hexo generate

    hexo deploy 这一步会上传到github，需要输入github用户名与密码

  6. 访问Username.github.io，来查看你的页面吧。
